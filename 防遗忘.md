Markdown 速查表，你直接新建一个文件比如 `BLOG_WORKFLOW.md` 放在博客根目录里就行。

~~~md
# Hexo 博客发布防遗忘清单（Windows）

> 仓库：kychen-bit/kychen-bit.github.io  
> 网站：https://kychen-bit.github.io  
> 本地项目目录：C:\Users\Mechrevo\Desktop\blog\my-blog

---

## 0. 打开项目目录（每次先做）
```powershell
cd C:\Users\Mechrevo\Desktop\blog\my-blog
~~~

## 1. 新建文章

```powershell
hexo new "文章标题"
```

文章路径：

- `source/_posts/文章标题.md`

------

## 2. 写文章

- 用 Markdown 写正文
- 可选：在头部加分类/标签

示例 Front-matter：

```md
---
title: 文章标题
date: 2025-12-24 22:00:00
categories:
  - 技术
tags:
  - Hexo
  - GitHub Pages
---
```

------

## 3. 本地预览（强烈建议）

启动：

```powershell
hexo s
```

浏览器打开：

- [http://localhost:4000](http://localhost:4000/)

停止服务：

- 在终端按 `Ctrl + C`

------

## 4. 发布上线（固定三连）

```powershell
git add .
git commit -m "post: 文章标题"
git push
```

------

## 5. 查看是否发布成功

GitHub 仓库 → Actions → `Deploy Hexo to GitHub Pages`

- 看到 ✅ 绿色对勾 = 发布成功

网站访问：

- [https://kychen-bit.github.io](https://kychen-bit.github.io/)

------

## 6. 修改已发布文章（改完再三连）

```powershell
git add .
git commit -m "fix: update post"
git push
```

------

## 7. 图片怎么放（推荐）

放到：

- `source/images/xxx.png`

Markdown 引用：

```md
![图注](/images/xxx.png)
```

------

## 8. 常见故障速查

### A) npm / hexo 命令不可用

```powershell
node -v
npm -v
hexo -v
```

### B) 本地能看，线上 404 / 样式乱

检查 `_config.yml`：

```yml
url: https://kychen-bit.github.io
root: /
```

改完后再：

```powershell
git add .
git commit -m "fix: config"
git push
```

### C) Actions 失败

- GitHub → Actions → 点红色那次 run → 复制报错日志来排查
